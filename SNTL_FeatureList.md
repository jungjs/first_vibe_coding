# SNTL 통합 물류 플랫폼 — 기능 리스트

> 총 144개 기능 항목 | L1(대분류) ~ L4(세부기능) 4단계 트리 구조 | 최종 업데이트: 2026-04-16

---

## Phase 별 기능 요약

| Phase | 도메인 | 주요 내용 |
|---|---|---|
| Phase 1 | 로그인/인증, 회원관리, 택배사관리 | JWT 인증, 개인/법인 회원가입·관리, 기초코드 |
| Phase 2 | 오더관리, 마스터오더관리, 창고관리 | 오더 등록·수정·삭제, Packing, 입출고 처리 |
| Phase 3 | 운송Tracking, 회계/청구, VOC관리 | 외부 API 연동, 청구서·세금계산서, VOC 처리 |
| Phase 4 | 시스템관리(메뉴/코드/권한/알림/통계), 고객지원 | 운영·관리 기능, 통계, QnA/FAQ/공지사항 |

## 1. 로그인/인증 (Phase 1)

> JWT 기반 Stateless 인증 방식. 전체 사용자 공통.

### 1.1 로그인

#### 1.1.1 ID/PW 로그인
- **사용자권한:** 회원/운영자/관리자
- **Phase:** 1
- 아이디와 비밀번호를 입력하여 로그인
- 성공 시 Access Token(30분)과 Refresh Token(7일) 발급
- 비밀번호는 BCrypt로 해싱하여 검증

#### 1.1.2 소셜 로그인
- **사용자권한:** 개인회원
- **Phase:** 1
- **비고:** 소셜 플랫폼 추후 확정
- Google·Kakao 등 OAuth2 소셜 계정으로 인증
- 최초 로그인 시 회원가입 연동 처리
- 개인회원 전용 기능

### 1.2 ID/PW 찾기

#### 1.2.1 ID 찾기
- **사용자권한:** 회원
- **Phase:** 1
- 이름과 휴대폰번호를 입력하고 SMS 인증번호를 통해 본인 확인 후 로그인 ID 화면에 표시
- 법인회원은 법인명 + 사업자번호로 확인

#### 1.2.2 PW 재설정
- **사용자권한:** 회원
- **Phase:** 1
- 이메일 또는 SMS 인증을 통해 본인 확인 후 새 비밀번호 설정
- 새 비밀번호 조건: 영문+숫자+특수문자 8자 이상

### 1.3 로그아웃
- **사용자권한:** 회원/운영자/관리자
- **Phase:** 1
- 로그아웃 요청 시 Redis에 저장된 Refresh Token 삭제
- Access Token을 블랙리스트에 등록하여 재사용 차단

## 2. 회원관리 (Phase 1)

> 개인회원·법인회원·관리자 계정의 등록, 수정, 조회, 탈퇴 전반 관리

### 2.1 개인회원

#### 2.1.1 회원가입

| 세부기능 | 설명 | Phase |
|---|---|---|
| 본인확인 | 이름·생년월일·휴대폰번호 입력, SMS 인증, 중복 가입 방지 | 1 |
| 개인정보 입력 | 로그인ID·비밀번호·이메일·주소 입력. ID·이메일 중복 불가 | 1 |
| 소셜 로그인 연동 가입 | OAuth2 인증 후 추가 정보 입력하여 계정 생성 | 1 |

#### 2.1.2 회원정보 수정
- **사용자권한:** 개인회원 | **Phase:** 1
- 이름·생년월일·휴대폰번호·주소·이메일 수정 가능
- 로그인ID 변경 불가

#### 2.1.3 회원 등급 조회
- **사용자권한:** 개인회원 | **Phase:** 1 | **비고:** 등급기준 추후 확정
- 등급 확인: BASIC / SILVER / GOLD / VIP
- 등급별 운송비 할인율 정보 함께 표시

#### 2.1.4 잔액 충전
- **사용자권한:** 개인회원 | **Phase:** 1
- 선불 방식으로 서비스 이용 잔액 충전
- 충전 금액은 청구서 결제에 사용 가능

#### 2.1.5 잔액 환불
- **사용자권한:** 개인회원 | **Phase:** 1
- 충전 잔액 중 미사용 금액 환불 요청
- 관리자 검토 후 진행, 청구서에 반영

#### 2.1.6 청구내역 조회
- **사용자권한:** 개인회원 | **Phase:** 1
- 청구서 목록 및 항목별 청구 이력 조회
- 기간·운송수단별 필터링, 청구서 PDF 출력 지원

#### 2.1.7 회원탈퇴
- **사용자권한:** 개인회원 | **Phase:** 1 | **비고:** Soft Delete
- 개인정보(이름·생년월일·연락처·주소·이메일) → NULL 처리
- 로그인ID만 보존 (Soft Delete)
- 탈퇴 후 동일 ID로 재가입 불가

#### 2.1.8 개인정보 활용동의
- **사용자권한:** 개인회원 | **Phase:** 1
- 서비스 이용 약관 및 개인정보 처리방침 동의 관리
- 필수 동의 항목 미동의 시 서비스 이용 제한

## 3. 오더관리 (Phase 2)

> 화물 운송 의뢰 단위인 오더를 등록·관리하는 핵심 기능

### 3.1 오더 기본관리

#### 3.1.1 오더 등록
- **사용자권한:** 회원/운영자/관리자 | **Phase:** 2
- 오더번호 자동 생성 (형식: `ORD-YYYYMMDD-NNNNN`)
- 초기 상태: `REGISTERED`

| 세부기능 | 설명 |
|---|---|
| 송하인 정보 입력 | 이름·국가코드·우편번호·주소·전화번호. 법인 시 사업자번호 추가 |
| 수하인 정보 입력 | 이름·국가코드·우편번호·주소·전화번호. 법인 시 사업자번호 추가 |
| 화물 정보 입력 | 품명·HS코드·원산지·PCS·PKGS·총중량(kg)·순중량(kg)·CBM 입력 |
| 특수화물 기재 | ① 위험물(IMDG코드) ② 냉동냉장(온도℃) ③ 고가품(단가/총액) ④ 중고품 |

#### 3.1.2 오더 수정
- **사용자권한:** 회원/운영자/관리자 | **Phase:** 2
- 송하인·수하인·화물 정보 수정 가능
- **입고(WAREHOUSED) 이후 수정 불가**

#### 3.1.3 오더 삭제
- **사용자권한:** 회원/운영자/관리자 | **Phase:** 2
- 송하인·수하인·화물 정보 전체 삭제
- **Packing 또는 입고 이후 삭제 불가**

#### 3.1.4 오더 조회
- **사용자권한:** 회원/운영자/관리자 | **Phase:** 2
- 오더 목록 및 상세 조회
- 조회 조건: 회원명·법인명·부서명·등록일자

---

### 3.2 서비스 관리

> 하나의 오더에 최대 4가지 서비스(AIR/SEA/CIR/CCL) 추가 가능

#### 3.2.1 서비스 추가

| 서비스 | 설명 | Phase |
|---|---|---|
| AIR(항운) | 출발/도착 국가코드 → 항공사·스케줄 선택. 현재일 기준 스케줄 자동 조회 | 2 |
| SEA(해운) | 출발/도착 국가코드 → 선사·스케줄 선택 | 2 |
| CIR(택배) | 도착국가코드 → 서비스 가능 택배사 선택. 선택 완료 시 운송장 출력 가능 | 2 |
| CCL(통관) | 도착국가코드 → 통관사 선택. 통관신고 처리와 연동 | 2 |

#### 3.2.2 서비스 수정
- 운송수단·스케줄 정보 수정
- 서비스 구분코드(AIR/SEA/CIR/CCL) 변경 불가 → 삭제 후 재등록

#### 3.2.3 서비스 삭제
- 입고 이후 처리 단계에서는 삭제 불가

#### 3.2.4 운항스케줄 조회
- AIR·SEA 서비스 추가 시 현재일 기준 운항 가능한 스케줄 목록 조회
- 출발지·도착지·운송사 조건으로 필터링

---

### 3.3 운송비용 관리

#### 3.3.1 운송비용 자동 계산
- **사용자권한:** 시스템 | **Phase:** 2
- 화물 총중량·CBM 기반으로 운송원가 산출 후 영업이익율과 회원할인율 적용

```
운송비 = 원가 + (원가 × 영업이익율%) − (원가 × (1 − 할인율%))
```

#### 3.3.2 Extra Charge 입력
- **Phase:** 2 | **비고:** 소수점 1자리
- 오더별 추가 비용 수동 입력 (예: 15.5)
- 회원·운영자: Extra Charge만 수정 가능
- 관리자: 운송원가·운송비·Extra Charge 모두 수정 가능

#### 3.3.3 운송비용 조회
- 회원: 운송비·Extra Charge만 조회
- 관리자: 원가 포함 전체 조회

| 4.5 마스터오더 조회 | 회원/운영자/관리자 | 2 | 목록 및 상세 조회. 포함 오더·서비스·Tracking 현황 확인 |

## 6. 운송 Tracking (Phase 3)

> 항운(AIR)·해운(SEA)·택배(CIR) 운송 현황 및 통관 신고 결과 조회
> **비고:** 외부 API 추후 확정

### 6.1 항운(AIR) Tracking

| 기능 | Phase | 설명 |
|---|---|---|
