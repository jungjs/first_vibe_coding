>

## 공통 레이아웃 규칙

### 전체 페이지 구조
```
┌─────────────────────────────────────┐
│  TopBar (로고 + 메뉴 + 사용자정보)    │ 48px
├──────────┬──────────────────────────┤
│ Sidebar  │  Main Content Area       │
│ (200px)  │                          │
│ 메뉴 목록 │  페이지 제목              │
│          │  ──────────────────────  │
│          │  검색/필터 영역            │
│          │  ──────────────────────  │
│          │  데이터 영역 (테이블/폼)   │
│          │                          │
└──────────┴──────────────────────────┘
│  Footer                             │ 40px
```

### 색상 팔레트
| 용도 | 색상 | HEX |
|------|------|-----|
| Primary (헤더, 중요요소) | Navy | `#1E3A5F` |
| Secondary (버튼, 링크) | Blue | `#2E86AB` |
| 배경 | Light Gray | `#F0F4F8` |
| 성공/확인 | Green | `#27AE60` |
| 경고/주의 | Orange | `#E67E22` |
| 오류/삭제 | Red | `#C0392B` |
| 강조 | Yellow | `#F39C12` |

### 공통 컴포넌트
- **검색바**: 화면 상단, 흰 배경, 회색 테두리
- **데이터 테이블**: Navy 헤더, 짝수행 연회색, hover 파란 배경
- **버튼**: Primary(파랑), Success(초록), Danger(빨강), Warning(주황)
- **태그/뱃지**: 둥근 모서리, 서비스/상태 구분에 사용
- **메타정보 바**: 화면ID, 사용자, Phase, API 표시 (헤더 아래)

## Section 1 — 로그인 / 인증

### SCR-001 로그인

**레이아웃**: 중앙 정렬 카드 (360px), 전체 화면 배경 `#F5F7FA`

**구성 요소**
```
┌────────────────────────────┐
│        SNTL  로고           │
│     통합 물류 플랫폼         │
│ ─────────────────────────  │
│  [아이디 입력 필드]          │
│  [비밀번호 입력 필드]  👁    │
│  [        로그인        ]   │  ← Primary 버튼 (100%)
│  ID찾기 | PW재설정 | 회원가입 │  ← 링크
│ ─────────────────────────  │
│       소셜 로그인            │
│  [ G Google ] [ K Kakao ]  │
└────────────────────────────┘
```

**비즈니스 규칙**
- 로그인 성공 → JWT Access Token(30분) + Refresh Token(7일) 발급
- 역할에 따라 이동: 관리자→대시보드(SCR-090), 일반회원→오더목록(SCR-020)
- 소셜 로그인 최초 접근 시 → 개인정보 추가 입력 후 계정 생성
- 비밀번호 표시/숨기기 토글 아이콘

**연결 화면**: SCR-002(ID찾기), SCR-003(PW재설정), SCR-004(개인가입), SCR-005(법인가입)

---

### SCR-002 ID 찾기

**레이아웃**: 탭 UI (개인회원 / 법인회원), 중앙 카드

**탭별 입력 필드**
| 탭 | 입력 필드 |
|----|---------|
| 개인회원 | 이름, 생년월일(YYYYMMDD), 휴대폰번호, SMS 인증번호 |
| 법인회원 | 법인명, 사업자등록번호 |

**버튼**: `인증번호 발송` (Green), `ID 찾기` (Primary)

**결과 표시**: 인증 완료 후 아이디 화면 표시 (중복 계정 시 복수 ID 표시)

---

### SCR-003 PW 재설정

**레이아웃**: 단계형 (인증 → 비밀번호 입력)

**입력 필드**: 이메일 또는 SMS 인증 선택 → 인증번호 → 새 비밀번호 → 비밀번호 확인

**규칙**: 영문 + 숫자 + 특수문자 8자 이상

---

### SCR-004 개인회원 가입

**레이아웃**: 3단계 스텝 인디케이터 + 단계별 폼

```
[STEP 1 본인확인] ▶ [STEP 2 정보입력] ▶ [STEP 3 완료]
```

**STEP 1 — 본인확인**
| 필드 | 타입 | 필수 |
|------|------|------|
| 이름 | Text | Y |
| 생년월일 | Text (YYYYMMDD) | Y |
| 휴대폰번호 | Tel | Y |
| SMS 인증번호 | Text (6자리) | Y |

- `인증번호 발송` (Green) → SMS 발송 → 인증번호 입력 → `인증 확인`
- 동일 휴대폰번호 중복가입 체크

**STEP 2 — 정보입력**
| 필드 | 타입 | 필수 | 규칙 |
|------|------|------|------|
| 로그인ID | Text | Y | 중복확인 버튼 |
| 비밀번호 | Password | Y | 영문+숫자+특수 8자↑ |
| 비밀번호 확인 | Password | Y | 일치 확인 |
| 이메일 | Email | Y | 중복 불가 |
| 국가코드 | Select | Y | |
| 우편번호 | Text | Y | |
| 주소 | Text | Y | |
| 상세주소 | Text | N | |

**STEP 3 — 완료**: 가입완료 메시지 + `로그인 화면으로 이동` 버튼

---

### SCR-005 법인회원 가입

**레이아웃**: 단일 폼 카드

| 필드 | 타입 | 필수 |
|------|------|------|
| 법인명 | Text | Y |
| 사업자등록번호 | Text | Y |
| 대표자명 | Text | Y |
| 법인 이메일 | Email | Y |
| 법인 연락처 | Tel | Y |
| 사업자등록증 | File Upload | Y |

**파일 업로드**: 드래그&드롭 영역, PDF/JPG/PNG, 최대 10MB

**가입 후 프로세스**: 가입 신청 → 관리자 심사 대기 → 승인/거부 알림 발송

## Section 2 — 회원관리

### SCR-010 마이페이지

**레이아웃**: 사이드바 + 정보 카드

**표시 항목**: 회원명, 이메일, 휴대폰, 등급(BRONZE/SILVER/GOLD/PLATINUM), 가입일, 최근 오더 건수

**버튼**: `회원정보 수정` (→SCR-011), `회원탈퇴` (→SCR-012)

---

### SCR-011 회원정보 수정

**수정 가능 필드**: 이메일, 연락처, 주소, 상세주소
**비밀번호 변경**: 현재 비밀번호 확인 후 새 비밀번호 설정 (별도 섹션)

---

### SCR-012 회원탈퇴

**레이아웃**: 확인 모달 또는 전용 페이지

**단계**: 탈퇴 안내문 확인 → 탈퇴 사유 선택(Select) → 본인 비밀번호 확인 → `탈퇴 확인` 버튼

**규칙**: Soft Delete (개인정보 컬럼 Null 처리, ID(PK) 보존)

---

### SCR-013 법인 마이페이지 / SCR-014 부서·멤버 관리

**SCR-013**: 법인 기본정보, 부서 목록, 멤버 수

**SCR-014 부서 관리**
- 부서 트리 목록 (부서명, 멤버 수, 관리 버튼)
- `부서 추가` → 폼 팝업 → 부서명 입력
- 멤버 초대: 이메일 입력 → 초대 메일 발송

---

## Section 3 — 오더관리

### SCR-020 오더 목록

**레이아웃**: 사이드바 + 검색 + 테이블 + 페이징

**검색 필드**
| 필드 | 타입 |
|------|------|
| 오더번호 | Text |
| 회원명 | Text |
| 서비스 | Select (전체/AIR/SEA/CIR/CCL) |
| 상태 | Select (전체/등록/창고입고/운송중/통관중/완료) |

**테이블 컬럼**: 오더번호 / 회원명 / 서비스(태그) / 운송구간 / 상태(태그) / 등록일 / 운송비 / 관리(상세링크)

**접근제어**: 회원은 본인 오더만 조회. 운영자/관리자는 전체 조회.

**버튼**: `+ 오더 등록` (Green, →SCR-021)

---

### SCR-021 오더 등록

**레이아웃**: 3개 섹션 폼 (송하인 / 수하인 / 화물정보)

**송하인 정보** (Navy 헤더)
| 필드 | 필수 |
|------|------|
| 송하인명 | Y |
| 연락처 | Y |
| 국가코드 | Y |
| 우편번호 | Y |
| 주소 | Y |
| 상세주소 | N |

**수하인 정보** (Blue 헤더) — 동일 구조

**화물 정보** (Orange 헤더)
| 필드 | 필수 |
|------|------|
| 화물명 | Y |
| HS코드 | N |
| 총중량(kg) | Y |
| CBM | N |
| 포장수량 | N |
| 화물가액(USD) | N |
| 특이사항 | N |

**버튼**: `취소` (Red), `저장` (Primary), `저장 후 서비스 추가 →` (Green, →SCR-023)

**규칙**: 오더번호 시스템 자동생성 (ORD-YYYY-NNNNN)

---

### SCR-022 오더 상세/수정

**레이아웃**: 오더 헤더 + 3컬럼 정보 + 서비스 테이블

**오더 헤더**: 오더번호(굵게) + 상태 태그 + 서비스 태그 + 등록일

**3컬럼**: 송하인 정보 | 수하인 정보 | 화물 정보

**서비스 정보 테이블** 컬럼: 서비스유형 / 출발지 / 목적지 / 출발일 / 도착예정일 / 운송비 / 상태

**버튼**: `수정` (Orange), `삭제` (Red), `+ 서비스 추가` (Green)

**규칙**: **창고 입고 이후 수정·삭제 버튼 비활성화** (disabled 처리)

---

### SCR-023 서비스 추가

**레이아웃**: 4개 탭 (AIR / SEA / CIR / CCL) + 탭별 폼

**AIR 항운 입력 필드**
| 필드 | 필수 |
|------|------|
| 출발 공항(IATA) | Y |
| 도착 공항(IATA) | Y |
| 항공사 | N |
| MAWB번호 | N |
| HAWB번호 | N |
| 출발 예정일 | N |
| 도착 예정일 | N |
| 특이사항 | N |

**SEA 해운 입력 필드**: 출발항구, 도착항구, 선사, MBL번호, HBL번호, 선박명, 항차, 출항일, 입항예정일

**CIR 택배 입력 필드**: 택배사(Select), 운송장번호, 출발국가, 도착국가

**CCL 통관 입력 필드**: 통관신고번호, 담당 세관, 신고유형, 신고일

**규칙**: 한 오더에 최대 4가지 서비스 추가 가능

---

### SCR-024 운송비용 조회

**레이아웃**: 서비스별 비용 카드 + Extra Charge 입력

**표시 항목**
```
서비스 운송원가 : ₩XXX,XXX
영업이익율     : X%
회원 할인율    : X%
──────────────────────
계산된 운송비  : ₩XXX,XXX
Extra Charge  : [입력 필드] (소수점 1자리)
──────────────────────
최종 운송비    : ₩XXX,XXX
```

**계산식**
```
운송비 = 서비스운송원가
       + (서비스운송원가 × 영업이익율)
       - (서비스운송원가 × (1 - 회원별할인율))
```

## Section 4 — 마스터오더관리

### SCR-031 마스터오더 등록/패킹

**레이아웃**: 좌우 2분할

**좌측 — 마스터오더 기본정보**
| 필드 | 비고 |
|------|------|
| 마스터오더번호 | 자동생성, 읽기전용 |
| 운송 서비스 유형 | Select |
| 출발지 | Text |
| 목적지 | Text |
| 출발 예정일 | Date |
| 도착 예정일 | Date |
| 비고 | Textarea |

버튼: `마스터오더 생성` (Green)

**우측 — 오더 패킹**
- 오더 검색 (오더번호/회원명) + `검색` 버튼
- 검색 결과 테이블: 선택체크박스 / 오더번호 / 회원명 / 서비스 / 중량
- `+ 패킹 추가` (Green), `패킹 제거` (Red)
- 현재 패킹 현황 표시: "패킹된 오더: N개 / 총 중량: XXkg"
- `패킹 완료 저장` (Navy)

**규칙**: 패킹된 오더는 단독 수정·삭제 불가. 마스터오더 단위로 일괄 처리.

## Section 5 — 창고관리

### SCR-040 창고 입고 처리

**레이아웃**: 좌우 2분할

**좌측 — 바코드 스캔**
- 카메라/스캐너 스캔 영역 (파란 점선 테두리)
- 구분선 "— 또는 직접 입력 —"
- 오더번호 직접 입력 필드 + `조회` 버튼

**우측 — 입고 확인 정보**
- 자동 조회된 오더 정보 표시 (오더번호, 마스터오더, 회원명, 서비스, 화물명, 현재상태)
- 입고 담당자 메모 (선택 입력)
- `✓ 입고 처리 확인` 버튼 (Green, 전체 너비)

**규칙**: 입고 처리 → 오더 상태 `창고입고`로 변경 → 이후 수정·삭제 불가 처리

---

### SCR-041 창고 출고 처리

**레이아웃**: 좌우 2분할

**좌측 — 출고 대상 목록**
- 테이블: 선택체크박스 / 마스터오더번호 / 오더수 / 서비스 / 운송장출력상태
- `🖨 운송장 출력` (Blue), `출고 처리` (Green)

**우측 — 운송장 미리보기**
- 운송장 레이아웃 프리뷰 (발송인, AWB번호, 수취인, 내용물, 서비스 표시)
- 바코드 이미지 영역
- PDF로 다운로드 → MinIO에서 파일 제공

## Section 6 — 운송 Tracking

### SCR-050 AIR Tracking

**레이아웃**: 검색바 + 타임라인

**검색바**: AWB번호 또는 오더번호 입력 + `🔍 조회` + 결과 태그 표시 (서비스/구간/상태)

**타임라인 구성**
```
● (완료) 2026-04-10 09:00  화물 접수    ICN 창고 입고 완료
● (완료) 2026-04-11 14:30  출항         KE 123편 인천공항 출발
⟳ (진행) 2026-04-13 08:00  경유지 도착  ORD 공항 트랜짓 처리 중
○ (예정) 2026-04-14 예정   목적지 도착  LAX 공항 도착 예정
○ (예정) 2026-04-15 예정   통관         세관 통관 처리 예정
○ (예정) 2026-04-15 예정   배송완료     수취인 인도 예정
```

**아이콘 색상**: 완료=Green, 진행=Orange, 예정=Gray

**규칙**: 외부 API 연동, 5분 주기 자동 갱신, 갱신 실패 시 최종 조회 시간 표시

### SCR-051 SEA / SCR-052 CIR Tracking

SCR-050과 동일한 타임라인 UI. 연동 API만 상이.

| 화면 | 연동 대상 | 주요 이벤트 |
|------|---------|-----------|
| SCR-051 | 선사 API | 선적/출항/환적/입항/통관/배송 |
| SCR-052 | 택배사 API | 집화/배송시작/배송중/배송완료 |
| SCR-053 | 세관 API | 신고접수/심사중/통관완료/반송 |

## Section 7 — 회계/청구

### SCR-060 청구서 목록

**레이아웃**: KPI 카드 4개 + 검색 + 테이블

**KPI 카드** (4개 가로 배치)
- 총 청구금액 (Navy)
- 총 입금액 (Green)
- 미수금 (Red)
- 청구서 건수 (Blue)

**테이블 컬럼**: 청구서번호 / 회원명 / 오더수 / 청구금액 / 입금액 / 미수금 / 청구일 / 상태(태그) / 관리

**상태 태그**: `완납`(Green), `일부납`(Orange), `미납`(Red)

**버튼**: `+ 청구서 생성` (Green)

---

### SCR-061 청구서 상세

**레이아웃**: 청구서 헤더 + 항목 테이블 + 합계 + 입금 처리

**청구서 헤더**: 청구서번호, 회원명, 청구일, 납기일, 상태

**항목 테이블**: 오더번호 / 서비스 / 운송비 / Extra Charge / 합계

**하단 합계**: 소계 / 부가세(10%) / 총 청구금액

**버튼**: `입금 처리` (→SCR-062), `세금계산서 발행` (→SCR-063), `PDF 다운로드`

---

### SCR-062 입금 처리

**입력 필드**: 입금 금액, 입금 날짜, 입금 방법(Select), 메모

**규칙**: 입금금액 ≤ 미수금. 전액 입금 시 상태 `완납`, 일부 시 `일부납`.

---

## Section 10 — 관리자

### SCR-090 관리자 대시보드

**레이아웃**: KPI 4개 + 차트 2개 + 처리 필요 알림

**KPI 카드** (상단 4분할)
| 지표 | 색상 | 내용 |
|------|------|------|
| 이번달 오더 | Navy | 건수 + 전월 대비 증감률 |
| 이번달 수익 | Green | 금액 + 증감률 |
| 처리중 오더 | Orange | 건수 + 증감률 |
| 미수금 | Red | 금액 + 증감률 |

**차트 영역** (하단 3분할)
- 월별 오더/수익 현황 (묶음 Bar Chart)
- 서비스별 오더 비중 (수평 Bar 또는 Donut Chart: AIR 45% / SEA 28% / CIR 18% / CCL 9%)
- 처리 필요 알림 목록 (법인심사대기, 미처리VOC, 미납청구서, 처리중QnA)

---

### SCR-091 회원 목록/관리

**검색 필드**: 회원명/ID, 이메일, 회원유형(Select), 상태(Select)

**테이블 컬럼**: 회원ID / 회원명 / 유형(태그) / 이메일 / 가입일 / 등급 / 상태 / 관리

**유형 태그**: `개인`(Blue), `법인`(Navy)

**상태**: `정상`(Green), `정지`(Red), `심사중`(Orange, 클릭 시 →SCR-092)

**버튼**: `엑셀 다운로드` (Green)

---

### SCR-092 법인회원 심사

**레이아웃**: 좌우 2분할 (법인정보 | 첨부파일 뷰어)

**좌측 — 법인정보**
- 테이블 형식 표시: 법인명, 사업자번호, 대표자, 법인이메일, 신청일, 상태
- `✅ 승인` (Green), `❌ 거부` (Red)
- 거부 사유 Textarea (거부 선택 시 필수 활성화)

**우측 — 첨부파일**
- PDF 인라인 뷰어 또는 이미지 미리보기
- `⬇ 파일 다운로드` 버튼

**규칙**: 승인 → 법인계정 `ACTIVE` 상태 변경 + 이메일/앱 알림 자동 발송

---

### SCR-093 메뉴 관리

**레이아웃**: 좌측 트리 + 우측 상세 편집 폼

**트리 구조**: 최상위 메뉴 > 서브메뉴 (드래그&드롭으로 순서 변경)

**편집 필드**: 메뉴명(한/영), 아이콘, URL, 정렬순서, 사용여부(Toggle)

---

### SCR-094 코드 관리

**레이아웃**: 좌측 코드그룹 목록 + 우측 코드 목록

**코드그룹**: 그룹코드, 그룹명, 설명, 사용여부

**공통코드**: 코드값, 코드명(한/영), 정렬순서, 사용여부
